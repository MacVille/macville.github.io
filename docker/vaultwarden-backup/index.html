Vaultwarden Backup

18.10.2025·694 Wörter·4 min
Autor
MacVille

Vaultwarden ist eine schlanke, selbst gehostete Alternative zu Bitwarden – perfekt für alle, die ihre Passwörter lieber selbst unter Kontrolle haben.
Aber: Wer selbst hostet, trägt auch selbst Verantwortung. Wenn deine Datenbank oder dein Container hopsgeht, ist dein Tresor futsch.
Und genau deshalb braucht dein Vaultwarden ein sauberes Backup.

In diesem Beitrag zeige ich dir, wie du ein automatisiertes Backup deiner Vaultwarden-Daten auf deinem Unraid-System einrichtest – mit einem simplen, aber effektiven Bash-Skript.

🧩 Warum du Vaultwarden unbedingt sichern solltest
#

Vaultwarden speichert:

  • deine komplette Passwortdatenbank (SQLite oder PostgreSQL)
  • Konfigurationen, Zertifikate und Icons
  • 2FA-Daten (falls du OTP nutzt)

All das liegt im Docker-Volume unter /mnt/user/appdata/vaultwarden.

Wenn deine SSD stirbt oder ein Update fehlschlägt, sind diese Daten ohne Backup unwiederbringlich verloren.

💡 Merksatz: Kein Backup = kein Mitleid.

🧰 Voraussetzungen
#

  • Unraid-Server mit Docker
  • Vaultwarden als Container installiert
  • Zugriff auf die Unraid-Konsole oder SSH
  • Basiswissen in Bash

⚙️ Das Backup-Skript
#

Erstelle auf deinem Unraid-Server ein neues Skript, z. B. unter:
/boot/config/plugins/user.scripts/scripts/vaultwarden_backup/script

Füge folgenden Inhalt ein:

#!/bin/bash
# Vaultwarden Backup Script for Unraid (Docker)
# Autor: Stefan / DockerCraft

# === Konfiguration ===
BACKUP_DIR="/mnt/user/backups/vaultwarden"
CONTAINER_NAME="vaultwarden"
DATE=$(date +"%Y-%m-%d_%H-%M-%S")
ARCHIVE_NAME="vaultwarden_backup_$DATE.tar.gz"

# === Container stoppen ===
echo "📦 Stoppe Vaultwarden Container..."
docker stop $CONTAINER_NAME

# === Backup erstellen ===
echo "📂 Erstelle Backup..."
mkdir -p "$BACKUP_DIR"
tar -czvf "$BACKUP_DIR/$ARCHIVE_NAME"     /mnt/user/appdata/vaultwarden

# === Container wieder starten ===
echo "🚀 Starte Vaultwarden Container..."
docker start $CONTAINER_NAME

# === Alte Backups löschen (optional) ===
find "$BACKUP_DIR" -type f -mtime +7 -name "*.tar.gz" -exec rm {} \;

echo "✅ Backup abgeschlossen: $ARCHIVE_NAME"

Mach die Datei ausführbar:

chmod +x /boot/config/plugins/user.scripts/scripts/vaultwarden_backup/script

🧠 Wie das Script funktioniert
#

  1. Container stoppen – Verhindert, dass während des Backups Schreibzugriffe auf die Datenbank passieren.
  2. Daten sichern – Alle wichtigen Dateien aus /mnt/user/appdata/vaultwarden werden in ein komprimiertes .tar.gz gepackt.
  3. Container neu starten – Vaultwarden läuft danach wieder ganz normal.
  4. Alte Backups löschen – Automatisch werden alle Backups, die älter als 7 Tage sind, entfernt.

🕒 Automatisierung in Unraid
#

Damit du nicht jedes Mal manuell sichern musst, kannst du das Ganze mit dem User Scripts Plugin automatisieren:

  1. Öffne das Plugin User Scripts in Unraid.
  2. Wähle das Script vaultwarden_backup.
  3. Stelle die Ausführung z. B. täglich um 3 Uhr nachts ein.

So läuft dein Backup vollautomatisch im Hintergrund – ohne manuelles Eingreifen.

☁️ Bonus-Tipp: Offsite-Backup
#

Ein Backup auf dem gleichen Server hilft dir nicht, wenn die Kiste abbrennt oder die Platte stirbt.
Deshalb empfehle ich zusätzlich:

  • Rclone → für automatischen Upload zu Google Drive, Dropbox oder OneDrive
  • rsync über SSH → um Backups auf ein NAS oder einen anderen Server zu spiegeln
  • Externes USB-Laufwerk → für manuelles, aber physisch getrenntes Backup

Beispiel mit Rclone:

rclone copy /mnt/user/backups/vaultwarden remote:unraid_backups/vaultwarden --progress

🔁 Restore: So stellst du Vaultwarden aus dem Backup wieder her
#

Ein Backup ist nur so gut wie seine Wiederherstellung.
Also testen wir, ob du im Notfall auch wirklich wieder an deine Daten kommst.

📦 Schritt-für-Schritt Anleitung
#

  1. Vaultwarden-Container stoppen

    docker stop vaultwarden

  2. Backup auswählen und entpacken

    cd /mnt/user/backups/vaultwarden
ls -lh
tar -xzvf vaultwarden_backup_YYYY-MM-DD_HH-MM-SS.tar.gz -C /

    🔎 Tipp: Das Archiv enthält /mnt/user/appdata/vaultwarden.
    Beim Entpacken nach / werden also die originalen Vaultwarden-Daten wiederhergestellt.

  3. Dateirechte prüfen

    chown -R nobody:users /mnt/user/appdata/vaultwarden
chmod -R 770 /mnt/user/appdata/vaultwarden

  4. Container starten

    docker start vaultwarden

  5. Testlauf machen

    Öffne deine Vaultwarden-Weboberfläche und prüfe, ob alle Logins, Sammlungen und Anhänge verfügbar sind.

🧠 Tipp für Fortgeschrittene
#

Wenn du dein Backup lieber manuell prüfen möchtest, ohne gleich alles zu überschreiben:

mkdir /mnt/user/temp/vaultwarden_restore_test
tar -xzvf vaultwarden_backup_YYYY-MM-DD_HH-MM-SS.tar.gz -C /mnt/user/temp/vaultwarden_restore_test

So kannst du vorher reinschauen, was im Archiv steckt, bevor du etwas überschreibst.

🚨 Wiederherstellung testen!
#

Viele Admins machen den Fehler, Backups nie zu testen.
Einmal im Quartal solltest du in einer sicheren Umgebung (z. B. Test-VM oder Backup-Unraid-Server) die Wiederherstellung durchspielen.
Nur so weißt du, dass dein Backup wirklich funktioniert – bevor es zu spät ist.

✅ Fazit
#

Mit dem zusätzlichen Restore-Teil ist dein Vaultwarden-Backup nun vollständig abgesichert:

  • Automatische, versionierte Backups
  • Alte Sicherungen werden bereinigt
  • Restore ist sauber dokumentiert
  • Optionaler Offsite-Sync für maximale Sicherheit

Damit hast du dein Passwort-Manager-System auf Unraid nicht nur selbst in der Hand – sondern auch unter Kontrolle, wenn mal was schiefgeht. 💪

🔒 Selbst gehostet ist cool. Aber erst mit Backup ist es wirklich sicher.
