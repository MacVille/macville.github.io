Meine Docker Container

Du willst mehrere Container in deinem Homelab orchestrieren, ohne jedes Mal manuell Hand anzulegen? In diesem Beitrag zeige ich dir, wie du mit Docker Compose dein Setup automatisierst – sauber, modular und reproduzierbar.

⚙️ Voraussetzungen
#

Kategorie Beschreibung
OS / Umgebung Ubuntu 24.04 LTS / Debian 12
Docker Version 27.x
Kenntnisse Grundwissen in Linux und Docker
Ziel Funktionierendes Setup mit Docker Compose

💡 Tipp: Wenn du Docker noch nicht installiert hast, schau dir zuerst meinen Artikel „Docker Installation auf Ubuntu in 5 Minuten“ an.

🚀 Schritt 1: Vorbereitung
#

Kurze Erklärung, warum dieser Schritt wichtig ist.

sudo apt update && sudo apt install docker-ce docker-compose-plugin -y

🔎 Erklärung:
Hier installierst du die aktuelle Docker-Version inklusive Compose-Plugin.
Verwende immer die offizielle Paketquelle, um Versionskonflikte zu vermeiden.

🧩 Schritt 2: Projektstruktur anlegen
#

Zeig, wie du dein Setup organisierst.
Verwende klare Beispiele, damit der Leser es 1:1 nachbauen kann.

mkdir -p ~/homelab/nginx && cd ~/homelab/nginx
touch docker-compose.yml

version: "3.9"
services:
  web:
    image: nginx:latest
    ports:
      - "8080:80"

💡 Pro Tipp:
Lege jedes Service-Setup in einen eigenen Ordner – das macht Updates und Backups später viel einfacher.

🧠 Schritt 3: Testen und Troubleshooting
# 

docker compose up -d
docker ps

Häufige Fehler & Lösungen:

Problem Ursache Lösung
Port already in use Port 8080 belegt Ändere den Port z. B. auf 8081
Container startet nicht YAML-Fehler Überprüfe Einrückungen (2 Leerzeichen pro Ebene)
Keine Logs sichtbar Container stürzt sofort ab Führe docker compose logs -f aus
